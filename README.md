# Proyecto Final de Carrera TDyPVJ

Bienvenido al repositorio oficial del **Proyecto Final de Carrera** de la Tecnicatura en Diseño y Programación de Videojuegos (TDyPVJ) de la Universidad Nacional del Litoral (UNL).

Este repositorio contiene la plantilla base del **Game Design Document (GDD)** que deberá ser utilizada y completada por los equipos para presentar la documentación de sus proyectos finales.

## 📄 Contenido del Repositorio

- `GDD.md`: Es la plantilla principal del Game Design Document estructurada con los requerimientos mínimos de la cátedra.

---

## 📌 Control de Versiones del Documento

El GDD es un documento vivo que cambiará a lo largo del desarrollo. Para llevar un control claro en el archivo `GDD.md`, utilizaremos un sistema de versionado numérico de tres partes **(X.Y.Z)**, por ejemplo `Versión: 1.2.0`:

- **X (Versión Mayor):** Modifica este primer número si hay un cambio drástico en la base del juego (ej. cambio de género, reestructuración completa de las mecánicas principales, reinicio del proyecto).
- **Y (Versión Menor):** Modifica el segundo número cuando agregues contenido nuevo e importante (ej. se añade un nuevo nivel al GDD, se documentan nuevos enemigos, o cambian mecánicas secundarias).
- **Z (Parche / Revisión):** Modifica el tercer número para correcciones menores (ej. corrección de errores tipográficos, actualización de imágenes, ajustes de redacción).

## 🛠️ Cómo hacer una copia de este proyecto para tu equipo

Para utilizar esta plantilla **NO debes solicitar acceso de edición** a este repositorio. Debes crear tu propia copia siguiendo uno de estos dos métodos:

### Método 1: Hacer un Fork (Recomendado)
Hacer un "Fork" creará una copia exacta de este repositorio en tu propia cuenta de GitHub, donde tendrás control total para editar y actualizar tu GDD.

1. Asegúrate de haber iniciado sesión en tu cuenta de GitHub.
2. Ve a la parte superior derecha de esta página.
3. Haz clic en el botón **`Fork`** (Bifurcar).
4. Selecciona tu cuenta como destino.
5. ¡Listo! Ahora tienes tu propia versión del proyecto para trabajar con tu equipo.

### Método 2: Descargar los archivos localmente
Si prefieres no usar un repositorio de Git por el momento, puedes simplemente descargar los archivos:

1. Ve a la parte superior de esta página y busca el botón verde que dice **`<> Code`**.
2. Haz clic en él y selecciona **`Download ZIP`**.
3. Descomprime el archivo en tu computadora y abre el archivo `GDD.md` con cualquier editor de texto o software compatible con Markdown (como Obsidian, Typora o Visual Studio Code).
